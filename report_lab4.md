# Отчёт по лабораторной работе №4

## Тема
Реализация основной функциональности приложения (этап 2 из 3).

## 1. Краткое описание интерфейсного решения
На этапе ЛР4 интерфейс, разработанный ранее, сохранён и дополнен рабочей интеграцией с backend. Все основные экраны продолжают функционировать как единая система: пользователь может получать данные по рынку, управлять ордерами, отслеживать изменения портфеля и просматривать историю операций.

Взаимодействие реализовано через REST API. Отображение данных организовано с учётом основных пользовательских состояний: загрузка, успешный результат, ошибка.

## 2. Пользовательские сценарии
1. Получение списка рыночных инструментов и текущих цен.
2. Создание ордера на покупку/продажу через экран торговли.
3. Обновление состояния портфеля после выполнения операции.
4. Просмотр истории ордеров.
5. Получение понятного сообщения об ошибке при некорректных параметрах ордера.

## 4. Описание реализованной части приложения
В рамках ЛР4 выполнено:
- реализация бизнес-логики обработки ордеров (покупка/продажа);
- реализация слоя доступа к данным через отдельные репозитории;
- интеграция UI и backend API;
- изоляция архитектурных слоёв (interface, business logic, data);
- обработка состояний загрузки, успеха и ошибок;
- тестирование базовых API-сценариев.

Источник данных организован в виде локального хранилища с мок-инициализацией, что соответствует требованиям этапа и позволяет стабильно демонстрировать работу функционала без внешних зависимостей.
