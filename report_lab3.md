# Отчёт по лабораторной работе №3

## Тема
Проектирование пользовательского интерфейса и начальный этап реализации приложения.

## 1. Краткое описание интерфейсного решения
В рамках ЛР3 спроектирован интерфейс учебного веб-приложения для моделирования криптотрейдинга. Выбрана структура из пяти основных экранов: дашборд, рынок, торговля, портфель и история ордеров. Экранные формы и навигация ориентированы на базовый пользовательский путь: просмотр данных рынка, создание ордера, контроль состояния портфеля и анализ выполненных операций.

Интерфейс реализован как SPA-приложение, что обеспечивает быстрые переходы между разделами без перезагрузки страницы. На данном этапе основной акцент сделан на корректной компоновке экранов, согласованности UI-сценариев и подготовке каркаса для последующей интеграции бизнес-логики.

## 2. Пользовательские сценарии
1. Пользователь открывает дашборд и получает сводную информацию по системе.
2. Пользователь переходит в раздел рынка и просматривает доступные инструменты.
3. Пользователь открывает экран торговли, выбирает инструмент и параметры ордера.
4. Пользователь переходит в раздел портфеля для просмотра текущих позиций.
5. Пользователь открывает историю ордеров для контроля предыдущих действий.

## 4. Описание реализованной части приложения
На этапе ЛР3 реализовано:
- базовая навигация между основными экранами;
- каркас (layout/skeleton) каждого ключевого экрана;
- первичная архитектурная структура backend по слоям MVC;
- подготовка API-слоя для последующей интеграции с frontend.

Функциональность намеренно оставлена неполной в соответствии с требованиями ЛР3, так как цель этапа — корректная архитектурная основа и соответствие интерфейса разработанному макету.
